# 架构与代码

## 项目结构

```

  constants.py      坐标系、棋子编码、九宫/半场判定
  board.py          棋盘状态、初始布局、显示
  movegen.py        走法生成、将军检测、白脸将、合法走法过滤
  game.py           对局状态(push/pop、终局判定、重复检测、长将判负)
  encoding.py       动作空间编码(2550 维固定动作表)
  state_encoder.py  状态张量编码(15×10×9,当前走方视角)
  network.py        策略-价值 ResNet
  evaluator.py      网络推理封装(单个/批量,供 MCTS 调用)

  selfplay.py       自我对弈,生成训练样本
  parallel_selfplay.py  多进程并行自我对弈(生产者-消费者)
  train.py          replay buffer(含持久化)+ 损失 + 训练步
  arena.py          评估门控:新旧网络对弈按得分率决定晋级
  pipeline.py       串行训练循环 + save/load/prune checkpoint
  pipeline_parallel.py  并行训练循环(多 worker,正式训练用)
  pipeline_ddp.py   多卡数据并行训练(torchrun + DDP)
  distributed.py    DDP 进程组初始化/rank 探测辅助
  metrics.py        训练指标记录(CSV,可选 TensorBoard)
  move_notation.py  中文记谱解析(炮二平五 -> 坐标)
  iccs_notation.py  ICCS 坐标记谱解析(C3-C4 -> 坐标)
  pgn_to_json.py    中文记谱 PGN -> 预训练 JSON
  pretrain.py       人类棋谱监督预训练(小数据,内存数组)
  pretrain_stream.py  大规模流式/分片预训练(大数据)
  encode_dataset.py 多进程并行预编码棋谱为磁盘分片
  play.py           命令行对弈接口
  webui.py          Web 对弈服务(标准库 http.server)
  webui_page.py     Web 前端页面(canvas 棋盘 + 交互 JS)
tests/              单元测试(87 项)
scripts/            部署/训练/对弈/备份自动化(见 docs/02)
data/               人类棋谱(all_games.json 等,见 docs/04)
```

## 已实现的工程化能力

1. 依赖锁定(requirements.txt 固定版本)
2. MCTS 批量推理(virtual loss)+ 树复用
3. 评估门控(arena.py 与 --gating,仅串行 pipeline)
4. replay buffer 持久化(--resume 一并恢复)
5. DDP 多卡数据并行(pipeline_ddp.py;单卡环境只验证了 world_size=1 退化路径)
6. 训练指标落盘(metrics.csv,可选 TensorBoard)
7. checkpoint 清理(--keep-checkpoints N)
8. 长将判负(game.py 的 _perpetual_check_loser;长捉仍按重复和棋)
9. 人类棋谱监督预训练(中文记谱 + ICCS 双解析,流式/分片大数据)
10. 全自动化部署/训练/备份脚本

## 设计要点

- **坐标**:10 行 × 9 列,`(row, col)`,row 0 红方底线。棋子正负整数编码。
- **动作空间**:2550 维固定表,随机对弈验证完整性。
- **状态编码**:当前走方视角(轮走方棋子始终前 7 平面)。
- **价值约定**:网络与 MCTS 节点价值均为"该节点轮走方"视角,回传逐层取反。
- **ICCS 坐标**:字母 A-I → col 0-8,数字 0-9 → row 0-9。
  例:`H2-E2` = 炮二平五 = `(2,7)→(2,4)`。
