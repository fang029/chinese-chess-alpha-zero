# 训练流程(自我对弈)

## 一句话

`pipeline_parallel.py` 多 worker 并行自我对弈生成对局,主进程消费样本训练,
每轮发布新权重给 worker 热重载。自我对弈是吞吐瓶颈。

## 启动(服务器)

```bash
ssh <your-host>
screen -dmS train bash ~/board/scripts/train.sh        # 全新训练
RESUME=1 screen -dmS train bash ~/board/scripts/train.sh  # 断点续训
```

挂起会话:在 screen 里按 **Ctrl+A** 再按 **D**(别按 Ctrl+C,那会停训练)。
回来看:`ssh <your-host>` 然后 `screen -r train`。

## train.sh 当前配置(A10 + 32 核)

```
pipeline_parallel --iterations 1000 --games-per-iter 24 --workers 28
  --simulations 200 --channels 64 --blocks 6 --batch-size 512
  --train-device cuda --worker-device cpu --max-moves 120
  --queue-size 128 --min-buffer 1000 --keep-checkpoints 5
```

参数要点:
- `--workers 28`:32 核留几个给主进程/系统。换 64 核机器时调到 ~60。
- `--channels 64 --blocks 6`:小网络。满规模 128×10 在 CPU 自对弈太慢。
  **续训时网络形状必须一致**,否则 load_state_dict 报 size mismatch。
- `--max-moves 120`:冷启动期随机网络爱走长和棋,限步避免单局过慢。
- `--games-per-iter 24`:每轮局数,设小让反馈更快。

## 怎么看进度(关键:看 metrics.csv,不要看 train.log)

```bash
ssh <your-host> "cat ~/board/checkpoints/metrics.csv"
```

每行一轮:`iteration, loss_total, loss_policy, loss_value, 红胜, 黑胜, 和,
均步, buffer, collect_sec, train_sec`。**loss_total 长期下降 = 在学**。

> train.log 经 tee 管道有块缓冲、长时间不刷新,看着像卡死实则在跑。
> metrics.csv 和 checkpoints 目录文件时间才是实时真相。

GPU 是否在用(收集对局阶段 0% 正常,只在每轮训练那几秒跳起来):
```bash
ssh <your-host> "nvidia-smi"
```

## 重要:GPU 利用率低是正常的,不是浪费

瓶颈是自我对弈,它是 batch=1 小推理,GPU 帮不上(实测 GPU 每步 2490ms vs
CPU 4456ms,只快 1.8 倍)。**加速靠加 CPU 核数,不是加/换 GPU**。要让 GPU
真正跑满需改造"批量推理服务"(几百行,暂未做)。详见 docs/05。

## 已知问题:纯随机冷启动会停滞

之前从随机权重直接自我对弈,loss 在 1.25 触底后**回升到 1.5**,且几乎全和棋
——网络学不出棋力。**解决办法是先做人类棋谱预训练**(docs/04),用 `--resume
checkpoints/pretrained.pt` 作为正式训练起点。

## 操作:停止训练

```bash
ssh <your-host> "screen -S train -X quit; pkill -9 -f pipeline_parallel"
# SSH 可能因负载高返回 255,重连确认:
ssh <your-host> "ps aux | grep -c '[p]ipeline_parallel'"   # 0 = 已停
```

## 换 64 核纯 CPU 服务器(用户已定方案)

**目标**:能下、不乱走、有基本棋力。这个目标 64×6 网络 + 64 核 CPU 足够,
不需要大网络或批量推理服务。

**机器**:云端 64 核纯 CPU 计算型(如 `ecs.c7.16xlarge`,**无 GPU**)。
自弈只吃 CPU,64 核比 32 核吞吐翻倍,且省掉 A10 的钱。

**脚本**:`scripts/train_cpu.sh`(已备好),关键参数:
```
--workers 60 --simulations 400 --channels 64 --blocks 6
--games-per-iter 50 --max-moves 200
--train-device cpu --worker-device cpu
--resume checkpoints/pretrained.pt --keep-checkpoints 5
```
- `--workers 60`:64 核留 4 个给主进程/系统。
- `--simulations 400`(从 200 提高):搜索加深 = "不乱走"的关键。预训练让网络
  会走,加深搜索让它走得准。
- `--resume pretrained.pt`:从预训练模型接续,不从零(脚本默认逻辑:有
  pretrained.pt 就用它起步;`RESUME=1` 则接 latest.pt 续训)。

**启动**:
```bash
ssh <your-host>
screen -dmS train bash ~/board/scripts/train_cpu.sh
```

**预期**:预训练已让网络会开局应招,纯 CPU 自弈强化几天即可达到"不乱走、
能对弈"。看进度同上(metrics.csv)。

**换机部署流程**:旧 GPU 机 `bash scripts/fetch.sh` 拉回 pretrained.pt → 开新
64 核 CPU 机 → `bash scripts/deploy.sh` 部署 → `bash scripts/restore.sh` 传回
pretrained.pt → 服务器 `screen -dmS train bash scripts/train_cpu.sh`。
