# 预训练流程(人类棋谱监督)

## 为什么要预训练

纯随机自我对弈冷启动会停滞(loss 触底回升、全和棋)。先用人类棋谱让网络
学到基本棋感(模仿人类走法 + 学对局结果),再转自我对弈强化。这是冷启动加速。

## 数据

- 来源:CGLemon 收集的 ICCS 棋谱(用户从 Google Drive 下载,放本地 `data/`)。
  - `WXF-41743games.pgns`(世界象棋联合会,4.2 万局)
  - `dpxq-99813games.pgns`(东萍,10 万局)
- 合并解析:`xiangqi/iccs_notation.py` 的 `parse_iccs_pgn` → `data/all_games.json`
  (14.16 万局,只存 moves+result,105MB)。ICCS 解析 100% 成功率。
- 胜负标签齐全(红胜 ~5.4 万 / 黑胜 ~4 万 / 和 ~4.8 万),价值目标可学。
- 约 1160 万样本。**全量编码成张量需数百 GB,不能一次性放内存** → 分片到磁盘。

## 完整流程(三步)

### 第 1 步:合并棋谱为 JSON(已完成,若重来)

```bash
# 本地,用 iccs_notation.parse_iccs_pgn 解析两个 pgns 合并
# 产物 data/all_games.json,已传服务器 ~/board/data/
```

### 第 2 步:并行编码为磁盘分片

单进程编码全量要 7 小时/epoch 且每 epoch 重编码。改用多进程一次性编码到磁盘:

```bash
ssh <your-host>
cd ~/board
screen -dmS encode bash -c 'export OMP_NUM_THREADS=1; \
  python -u -m xiangqi.encode_dataset \
  --games data/all_games.json --notation iccs --out-dir data/encoded \
  --workers 30 --shard-samples 300000 2>&1 | tee encode.log'
```

- 30 进程并行,~每分钟 30 万样本,全量约 40 分钟。
- 输出 `data/encoded/shard_*.npz`,每片 30 万样本,全量 ~38 片 ~230GB
  (1TB 磁盘够)。

看进度 / 判断完成:
```bash
ssh <your-host> "tail -3 ~/board/encode.log"                          # 末行「[编码] 完成」
ssh <your-host> "ls ~/board/data/encoded/shard_*.npz | wc -l"         # 分片数
ssh <your-host> "ps aux | grep -c '[e]ncode_dataset'"                     # 0 = 结束
```

> **坑**:中途 kill 编码会留下半截损坏分片(np.load 报错)。训练前删掉最后
> 一个不完整分片:
> `ssh <your-host> "cd ~/board && python3 -c \"import numpy as np; np.load('data/encoded/shard_XXXX.npz')['zs']\" || rm data/encoded/shard_XXXX.npz"`

### 第 3 步:分片训练(GPU)

数据已编码好,训练只管喂 GPU,多 epoch 不重编码:

```bash
ssh <your-host>
cd ~/board
screen -dmS pretrain bash -c 'python -u \
  -m xiangqi.pretrain_stream --shards data/encoded \
  --channels 64 --blocks 6 --epochs 3 --batch-size 1024 --device cuda \
  --out checkpoints/pretrained.pt 2>&1 | tee pretrain.log'
```

看进度 / 判断完成:
```bash
ssh <your-host> "tail -5 ~/board/pretrain.log"      # 每分片打印 loss;末行「已保存」
ssh <your-host> "ls -la ~/board/checkpoints/pretrained.pt"
```

- `--channels 64 --blocks 6` **必须与正式训练一致**,否则 resume 报错。
- 预期 loss 跨 epoch 下降(policy loss 从 ~6 降向 ~2)。

## 完成后:接正式训练

```bash
# pretrained.pt 是正式训练的起点
ssh <your-host>
RESUME=1 # train.sh 默认 resume latest.pt;预训练首训需手动指定 pretrained.pt:
cd ~/board && screen -dmS train bash -c 'python \
  -u -m xiangqi.pipeline_parallel --resume checkpoints/pretrained.pt \
  ... (其余参数同 train.sh) 2>&1 | tee train.log'
```

或把 `pretrained.pt` 复制为 `latest.pt` 后用 `RESUME=1 bash scripts/train.sh`
(注意 pretrained.pt 的 iteration=0,会从第 0 轮开始)。
